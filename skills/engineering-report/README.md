# engineering-report

用于把实现计划、代码落点、测试覆盖和可观测性整理成工程文档的 skill，强调与真实代码保持对齐。

## 包含内容

- `SKILL.md`：完整的工程报告工作流与写作规范
- `agents/openai.yaml`：agent 展示名、简介和默认提示词

## 适用范围

- 实现前的工程计划
- 实现后的工程落地报告
- 代码改动后的工程文档同步
- 交付前的实现完整性检查

## 不负责的内容

- 方案推导长文
- 架构原则论证
- 设计取舍主文档
- ADR 正文编写

这类内容应优先交给 `design-report`，工程报告只做引用和对齐说明。
